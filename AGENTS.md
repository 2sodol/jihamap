# AGENTS.md

이 파일은 JihaMap 저장소 루트 전체에 적용되는 로컬 작업 규칙이다. 프론트엔드를 수정할 때는 추가로 [frontend/AGENTS.md](/Users/wooseok/Desktop/jihamap/frontend/AGENTS.md)를 함께 따른다.

## Build And Verify

변경한 스택에 맞춰 아래 명령을 모두 통과시켜야 한다.

```bash
cd frontend && npm run lint
cd frontend && npm run build
cd backend && ./gradlew test
cd backend && ./gradlew build
```

규칙:

- 프론트엔드만 바꾸면 프론트엔드 두 명령은 필수다.
- 백엔드만 바꾸면 백엔드 두 명령은 필수다.
- 계약이나 연동이 바뀌면 네 명령 모두 실행한다.

## Core Rules

- 검증 명령 통과 전에는 작업을 완료로 표시하지 않는다.
- 요청받은 범위를 넘는 디렉터리나 스택은 건드리지 않는다.
- 배포, 푸시, 외부 시스템 변경, 민감정보 접근은 승인 없이 수행하지 않는다.
- 포트 `43210`과 `43211`, 기본 로컬 연동 방식은 목표가 명시적으로 요구하지 않는 한 유지한다.
- 환경 변수, 실행 절차, 운영자 워크플로우가 바뀌면 관련 문서를 같은 작업 안에서 업데이트한다.
- 큰 구조 개편보다 최소한의 안전한 변경을 우선한다.

## Project Invariants

- 프론트엔드는 모바일 우선 UI를 유지하고, `NEXT_PUBLIC_VWORLD_API_KEY`가 없어도 베이스맵이 동작해야 한다.
- 백엔드 공개 API는 기본적으로 `/api/v1` 네임스페이스 아래에 둔다.
- `GET /api/v1/system/status`는 로컬 프론트엔드에서 접근 가능해야 하며, CORS 로컬 오리진 정책을 깨뜨리면 안 된다.
- 명시적 합의 없이 public API 응답 구조를 깨거나 프론트엔드 소비 코드를 분리된 상태로 두지 않는다.
- 새 의존성 추가, 스키마/스토리지 도입, 외부 API 통합은 목표 문서에 이유와 범위를 적었을 때만 허용한다.

## Approval Required

아래 항목은 자동 실행 대상이 아니다.

- 배포 또는 운영 환경 변경
- 외부 API 키, 비밀 값, 계정 접근
- 네트워크를 통한 외부 시스템 상태 변경
- 대량 삭제나 되돌리기 어려운 파일 조작
- 워크스페이스 외부 쓰기
